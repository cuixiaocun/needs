## 设计定位

### 核心理念

**"让农户和小商户能轻松使用"**

- ✅ 圆润、友好的视觉风格
- ✅ 所有可交互元素都有明确的文字标签（除返回按钮）
- ✅ 充分的白空间和清晰的信息层级
- ✅ 避免复杂动画，保持操作流畅
- ✅ 支持字体大小调节（默认标准模式，可通过系统设置调整）

### 应用范围

- **iOS App**（农户/买家/工作人员）
- **Android App**（农户/买家/工作人员）
- **Web 版本**（可选，参考此设计规范）

三个角色使用同一套 App，通过登录时的角色标识进行区分，在导航和功能上有所差异但整体 UI 风格统一。

## 排版规范

### 字体选择

**系统字体**（推荐，无需引入自定义字体）

- **iOS**：SF Pro Display（系统默认）
- **Android**：Roboto（系统默认）
- **Web**：SF Pro Display / Roboto / 系统字体栈

**优点**：渲染清晰、加载快、系统集成度高

### 字体大小规范

**默认标准模式**（用户可通过系统设置调节，影响整个应用）

| 元素 | 大小 | 行高 | 字重 | 用途示例 |
|------|------|------|------|---------|
| **大标题** | 28sp | 1.2 (33.6sp) | 600 Bold | 页面主标题（如"订单详情"） |
| **标题** | 20sp | 1.3 (26sp) | 600 Bold | 卡片标题、大段落标题 |
| **主文本** | 16sp | 1.5 (24sp) | 400 Regular | 正文、按钮文字、输入框 |
| **副文本** | 14sp | 1.5 (21sp) | 400 Regular | 辅助说明、次要信息 |
| **说明文本** | 12sp | 1.4 (16.8sp) | 400 Regular | 小提示、日期时间、标签 |
| **极小文本** | 10sp | 1.2 (12sp) | 400 Regular | 版本号、法律声明等极少使用 |

### 字重分配

| 场景 | 字重 | 说明 |
|------|------|------|
| **标题** | 600 (Semibold) | 标题、强调信息 |
| **正文** | 400 (Regular) | 主要内容、输入框 |
| **辅助** | 400 (Regular) | 副文本、说明 |

## 组件设计

### 1. 按钮组件

#### 主操作按钮

```
┌─────────────────────────────┐
│  [🔑] 登录                  │
└─────────────────────────────┘

设计规范：
- 背景色：#2E7D32（深绿）
- 文字色：#FFFFFF（白色），16sp
- 高度：48-56dp（推荐 56dp）
- 圆角：12dp（圆润但不过度）
- 左右内边距：24dp
- 前缀图标：18×18dp，白色
- 触碰反馈：涟漪动画（Material Design 风格）
- 禁用状态：背景 #BDBDBD，文字 #757575，无涟漪
```

#### 次操作按钮

```
┌─────────────────────────────┐
│  [↗] 分享                   │
└─────────────────────────────┘

设计规范：
- 背景色：#FFFFFF（纯白）
- 边框：1dp #2E7D32（深绿）
- 文字色：#2E7D32（深绿），16sp
- 高度：48-56dp
- 圆角：12dp
- 左右内边距：24dp
- 前缀图标：18×18dp，深绿色
- 触碰反馈：背景变浅（#F1F8E9）+ 涟漪
```

#### 文本按钮（链接式）

```
[取消] [了解更多] [查看全部]

设计规范：
- 背景：透明
- 文字色：#2E7D32（深绿），16sp
- 字体样式：normal
- 触碰反馈：文字下划线 + 背景变浅
- 不需要图标（或仅在右侧加小箭头 →）
```

#### 按钮在表单中的应用

```
┌─────────────────────────────┐
│ 账号      [____________]    │
│ 密码      [____________]    │
│                             │
│  [🔑] 登录    [取消]        │
│              （左右排列）    │
└─────────────────────────────┘

规则：
- 按钮组间距：12dp
- 主操作按钮在左，次操作在右
- 或主操作按钮宽度 100%，次操作在下方
```

## Logo 设计

### Logo 规范

#### 设计概念

**"果蔬麦融合"** — 在圆角方形容器内，三个元素分别放在不同象限，清晰可辨但整体协调。

#### 设计结构

```
┌─────────────────────┐
│                     │
│  🍎    |    🌾     │
│  果    |    麦      │
│        |           │
├────────┼────────────┤
│        |           │
│    蔬  |           │
│   🥬   |           │
│                     │
└─────────────────────┘

或简洁版：
    🍎
┌─────────┐
│ 🥬  🌾  │
└─────────┘
```

#### 配色方案

| 元素 | 图标 | 颜色值 | 十六进制 | 说明 |
|------|------|--------|---------|------|
| 果 | 🍎 | 红色/橙色 | `#E74C3C` 或 `#FF6B35` | 新鲜、活力 |
| 蔬 | 🥬 | 深绿 | `#2E7D32` | 主品牌色，与应用统一 |
| 麦 | 🌾 | 金黄色 | `#D4AF37` 或 `#F4C430` | 丰收、稳定 |

#### Logo 背景

- **彩色版**：#FFFFFF（纯白）或 #FAFAFA（极浅灰），用于 App 图标、页面 Header
- **单色版**：#2E7D32（深绿），用于黑白打印、简化显示或反色场景

#### 应用尺寸

| 用途 | 尺寸 | 说明 |
|------|------|------|
| **App 图标** | 108×108dp（192×192px 高分辨率） | iOS / Android 应用图标 |
| **页面 Header** | 48×48dp | 顶部导航栏 Logo |
| **Tab 标签** | 32×32dp | 底部导航 Tab 图标（如需） |
| **启动页** | 256×256dp | 应用启动 Splash Screen |
| **网站** | 200×200dp 或 100×100dp | 官网、小票、文档 |

#### Logo 保护区

Logo 周围应保留最少 8dp 的空白区域，不放置其他元素。

## 导航结构

### 应用流程

```
启动页 (Splash Screen, 1-2秒)
    ↓
登录/注册页面 (Login / Register)
    ↓
角色选择（或根据账号自动识别）
    ├─ 农户 (Farmer)
    ├─ 买家 (Buyer)
    └─ 工作人员 (Staff)
    ↓
主应用 (Main App with Bottom Navigation)
    ├─ 首页 (Home)
    ├─ 订单 (Orders)
    ├─ 钱包 (Wallet)
    └─ 我的 (Profile)
```

### 页面层级

**深度一般不超过 3 层**（重要原则，保证易用性）

```
首页
  ├─ 首页 Tab
  └─ 详情页 → 详情操作

订单
  ├─ 订单列表 Tab
  ├─ 订单详情 → 订单操作（支付/取消等）
  └─ 创建订单 → 步骤 1,2,3 → 确认

钱包
  ├─ 钱包总览 Tab
  ├─ 交易记录
  ├─ 充值
  └─ 提现

我的
  ├─ 个人资料
  ├─ 实名认证
  ├─ 设置
  └─ 关于
```

## 布局原则

### 间距规范

**基础间距单位：8dp**

| 间距 | 数值 | 用途 |
|------|------|------|
| **xs** | 4dp | 相邻文字间距、小元素间 |
| **s** | 8dp | 列表项内间距、相邻组件间 |
| **m** | 12dp | 卡片内容间距、组间间距 |
| **l** | 16dp | 页面边距、大组件间距 |
| **xl** | 24dp | 页面大间距、分段间距 |
| **xxl** | 32dp | 页面顶部/底部安全距离 |

### 容器设计

```
┌────────────────────────────────────┐  ← 页面容器
│                                    │
│  ← 16dp (页面边距)                 │
│  ┌──────────────────────────────┐  │
│  │ 【卡片容器】                 │  │
│  │ ● 背景：#FFFFFF             │  │
│  │ ● 圆角：12dp                │  │
│  │ ● 边框：1dp #E0E0E0         │  │
│  │ ● 阴影：0.5dp 高度           │  │
│  │ ● 内边距：16dp              │  │
│  └──────────────────────────────┘  │
│  ↓                                  │
│  12dp (卡片间距)                    │
│  ↓                                  │
│  ┌──────────────────────────────┐  │
│  │ 【第二张卡片】               │  │
│  └──────────────────────────────┘  │
│                                    │
└────────────────────────────────────┘
```

### 白空间原则

- **页面顶部**：16dp 距离导航栏（不含状态栏）
- **页面底部**：32dp（避免被底部导航遮挡，或当没有内容时的余量）
- **卡片列表**：每张卡片外边距 16dp，卡片间距 12dp
- **充分留白**：不要让屏幕看起来过满，信息应该有呼吸感

### 响应式考虑

```
【小屏幕（3-4 英寸）】
- 字体：默认或更小
- 按钮高度：48dp
- 外边距：12dp
- 卡片间距：8dp

【中屏幕（4.5-6 英寸）】【推荐】
- 字体：标准 16sp
- 按钮高度：56dp
- 外边距：16dp
- 卡片间距：12dp

【大屏幕（6+ 英寸或平板）】
- 字体：标准或更大
- 按钮高度：56-64dp
- 外边距：24dp
- 卡片间距：16dp
- 最大内容宽度：可考虑 600-800dp
```

## 交互反馈

### 触碰反馈

| 交互 | 反馈方式 | 说明 |
|------|---------|------|
| **按钮点击** | 快速涟漪动画 + 轻微缩放（95%） | Material Design 涟漪，无延迟 |
| **卡片点击** | 阴影增强 + 背景变浅 | 不用涟漪，用背景变化 |
| **输入框焦点** | 边框变深绿 + 阴影出现 | 边框从灰变绿，阴影 4px 深绿 20% |
| **Tab 切换** | 淡入淡出或滑动 | 1-2 帧过渡，无卡顿感 |

### 加载状态

```
┌─────────────────────────────────┐
│                                 │
│       [旋转圆圈动画]            │
│                                 │
│       加载中，请稍候...         │
│                                 │
└─────────────────────────────────┘

设计规范：
- 圆圈：48×48dp，边宽 4dp，#2E7D32（深绿）
- 文字：14sp #757575，圆圈下方 16dp
- 背景：遮盖层，黑色 20% 透明度（可选，不阻断用户交互时不需要）
- 动画：360° 旋转，2s 循环
```

```
● 文字：#C62828（深红），14sp
● 高度：48dp，圆角 8dp
● 停留时间：4-5 秒自动消失（用户可滑动关闭）
● 反馈：轻微弱震动
```

### 页面过渡

- **页面进入**：从右侧滑入（iOS）或淡入（Android），200-300ms
- **页面返回**：向左滑出（iOS）或淡出（Android），200-300ms
- **Modal 弹窗**：从下方滑入，200ms；关闭时滑出，200ms
- **避免**：过长的过渡动画（会让年长用户感到困惑）

## 无障碍设计

### 颜色对比度

所有文本与背景的对比度应符合 **WCAG AA** 标准：
- **普通文本**：最少 4.5:1 的对比度
- **大文本**（18sp+）：最少 3:1 的对比度

**检查清单**：
- ✓ 深绿 #2E7D32 + 白色 #FFFFFF → 对比度 6.48:1（通过）
- ✓ 中灰 #757575 + 白色 #FFFFFF → 对比度 4.54:1（通过）
- ✓ 浅灰 #BDBDBD + 白色 #FFFFFF → 对比度 2.13:1（不通过，仅用于禁用）

### 触碰目标

- **最小尺寸**：48×48dp（符合 Google Material Design 和 Apple HIG）
- **最小间距**：8dp（按钮间相邻最小距离）
- **文字按钮**：可通过增加内边距达到 48×48dp

### 文本标签

- **所有可交互元素都应有文本标签**
  - 按钮：带文字（除顶部返回按钮）
  - 图标：旁边或下方有文字说明
  - 输入框：上方有标签或占位符有说明
- **屏幕阅读器支持**：提供语义标签（Semantics widget）

### 字体大小调节

用户可通过系统设置调节应用的字体大小，影响所有文本：

```
系统设置：
  ├─ 小字号（-20%）
  ├─ 标准（默认）
  └─ 大字号（+20%-40%）
```

应用应支持 `.textScale` 的系统级别调节，不强制固定大小。

## 响应式设计

### 屏幕尺寸适配

| 设备类型 | 屏幕宽度 | 设计策略 |
|---------|---------|---------|
| **小手机** | 320-375dp | 单列布局，宽边距 12dp，字体可略小 |
| **标准手机** | 375-414dp | 单列布局，宽边距 16dp，标准字体（推荐） |
| **大手机** | 414-480dp | 单列或两列，宽边距 16-24dp，字体标准 |
| **平板** | 600dp+ | 两列/网格布局，宽边距 24-32dp，最大内容宽度 800dp |

### 方向适配

- **竖屏（Portrait）**：主要使用场景，优先设计竖屏
- **横屏（Landscape）**：支持但不强制，内容可能排列调整
  - 导航栏改为侧边栏（可选）
  - 内容宽度增加，可考虑两列布局

### Safe Area（安全区域）

```
【iOS 刘海屏 + Android 挖孔屏】
┌─────────────────────────────┐
│ 【系统状态栏/安全区域】      │  ← 不放置内容
├─────────────────────────────┤
│ ← 标题                  [⋮] │  ← 导航栏，避开安全区
├─────────────────────────────┤
│ 【页面内容开始】            │
│                             │
│                             │
├─────────────────────────────┤
│ [🏠] 首页 │ [📦] 订单 ...   │  ← 底部导航，避开安全区
└─────────────────────────────┘
```

**必须为以下区域留出安全距离**：
- 顶部：状态栏 + 刘海/挖孔（iOS 44dp, Android 32-48dp）
- 底部：系统导航栏（Android，通常 48dp）
