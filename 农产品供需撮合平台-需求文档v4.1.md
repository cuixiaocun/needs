## 一、产品概述

### 1.1 产品定位

> **农产品智能撮合交易所**：用 AI 撮合菜农和采购方，双方完全不联系，平台全程托管。农户将菜送到平台实体市场集散，买家自提或由平台代运送货，实现农产品 B2B 批发的暗池交易。

### 1.2 与传统模式的对比

| 维度 | 传统农批市场 | 一般撮合平台 | **本平台** |
|------|------------|------------|----------|
| **买卖双方接触** | 直接面对面 | 平台介绍后联系 | **完全不联系** |
| **价格谈判** | 当面议价 | 双方私聊 | **AI 自动撮合区间** |
| **货款流转** | 现金/转账 | 双方自行 | **平台担保托管** |
| **质量保障** | 看货决定 | 自行判断 | **实体市场验收 + 拍照 + 退货** |
| **物流** | 自己安排 | 自己叫车 | **买家自提 / 市场代运** |

### 1.3 核心价值

- **零跳单风险**：双方通过实体市场中转，物理不接触 + 暗池撮合
- **低信任成本**：资金担保 + 市场验收 + 图片存证 + 代理人仲裁
- **极致效率**：AI 自动撮合 + 拼单 + 代运拼车
- **价格透明**：明码标价，无中间盘剥
- **履约可控**：农户违约有兜底，买家可定制偏好

### 1.4 业务范围

- **业务类型**：B2B 批发（不是 B2C 零售）
- **卖方**：菜农、果农、合作社
- **买方**：超市采购、批发商、餐饮企业
- **商品**：蔬菜、水果等农产品
- **地区**：山东先试点（依托本地实体市场），后期 SaaS 复制

## 二、业务模式（核心）

### 2.1 暗池撮合 + 实体市场集散

```
┌─────────────────────────────────────────┐
│           AI 撮合引擎                    │
│   ┌──────────┐        ┌──────────┐     │
│   │  卖单池  │        │  买单池  │     │
│   │          │ ←撮合→ │          │     │
│   └──────────┘        └──────────┘     │
└─────────────────────────────────────────┘
         ↑                       ↑
    只和 AI 对话              只和 AI 对话
         ↑                       ↑
       农户 ──送货──┐   ┌──自提──> 买家
                  ↓   ↓
               ┌────────┐
               │ 实体市场 │
               │（集散）  │
               └────────┘
                  │
                  └──代运──> 买家
```

### 2.2 关键业务前提

1. **运营方拥有一个实体蔬菜批发市场**（物理场所）
2. **先撮合后送货（JIT 模式）**：撮合成功后农户才按订单备货并送达市场
3. **市场不持有常备库存**：市场不是商品所有者，只是撮合后的集散节点
4. **双路径交付**：买家挂单时选择自提或代运

### 2.3 双向挂单 + 区间匹配

**卖家挂单**：
```
商品：花生
数量：2000 斤
等级：一级
价格区间：最低 3.3 元 / 期望 3.5 元
产地：山东临沂
承诺送达市场时间：2026-04-15 10:00
图片：[3 张实物图片]
```

**买家挂单**：
```
商品：花生
数量：2000 斤
等级要求：一级及以上
价格区间：目标 3.4 元 / 最高 3.6 元
交付方式：[自提 / 代运]（必选）
  - 自提：无运费
  - 代运：填配送地址，平台预估运费
期望提货/送达时间：2026-04-15 14:00
```

**AI 撮合逻辑**：
```
商品、等级、数量、价格、时间、距离全部匹配
↓
成交价 = 价格区间交集的中间值
↓
匹配成功 → 自动成交
```

### 2.4 暗池 + 集散的双重隔离

| 层级 | 隔离方式 |
|---|---|
| **数据层** | 双方互相看不到对方身份、联系方式 |
| **物理层** | 农户送到市场，买家从市场提货/收货，双方在物理上永不碰面 |
| **沟通层** | 无 IM，只和 AI 对话 |
| **资金层** | 钱不走双方，只在平台账户流转 |

## 三、用户角色

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| **卖方（农户）** | 果农、菜农、合作社 | 快速卖出农产品 |
| **买方（采购方）** | 超市、批发商、餐饮企业 | 稳定货源，质量可靠 |
| **市场工作人员** ⭐ | 运营市场实体场所的工作人员（第一期合并为一个角色，负责收货/分级/出货/代运装车） | 高效处理订单流转 |
| **代理人** | 平台自有团队（2-3 人起步） | 处理撮合不了的复杂订单 + 紧急调货 + 争议仲裁 |
| **管理员** | 平台运营人员 | 监控数据、配置费率、处理纠纷 |
| **运输方（第三方）** | 货拉拉（第一期唯一对接） | API 接入，不在平台注册 |

## 四、核心业务流程

### 4.1 完整交易流程图（v4.1）

```
卖家进入 App → 和 AI 对话 → 挂单 → 进入卖单池
                                       ↓
                                   AI 撮合引擎
                                       ↓
买家进入 App → 和 AI 对话 → 选择自提/代运 → 挂单 → 进入买单池
                                       ↓
                                匹配成功（含价格、等级、时间、运输方案）
                                       ↓
                              通知双方（不知道对方是谁）
                                       ↓
                                买家付款 → 进入市场支付宝账户
                                       ↓
                       [分支 A] 农户备货 → 按时送达市场
                              ↓
                       市场工作人员扫码 → 过磅 → 目视分级 → 拍照
                              ↓
                       ┌────正常────┐
                       │   降级：通知买家确认差价  │
                       │   拒收：触发农户违约流程  │
                       └────────────┘
                              ↓
                          入库暂存
                              ↓
                       通知买家"货已到"
                              ↓
                    ┌──自提──┐     ┌──代运──┐
                    │提货码核验│    │呼叫货拉拉│
                    └──────┘      └──────┘
                              ↓
                         7 天异议期
                              ↓
                  ┌─无异议→自动确认→本周结算
                  └─有异议→代理人仲裁→退款/分账

                       [分支 B] 农户申请取消 / 超时未送货
                              ↓
                       进入农户违约兜底流程（见第 X 章）
```

### 4.2 三种异常分支的兜底
- **农户违约**：进入"第一段呼叫代理人 → 第二段 AI 调货 → 第三段自动取消"（详见第 X 章）
- **到货降级**：买家接受差价 or 拒收转违约流程
- **货物超卖/短缺**：按比例退款 or 拒收转违约流程

### 4.3 买家端详细流程（v4.1 新增）

#### 4.3.1 挂单阶段

```
买家打开 App → AI 对话引导挂单
  ↓
输入信息：
  - 商品：花生
  - 数量：2000 斤
  - 等级：一级及以上
  - 价格区间：目标 3.4 元 / 最高 3.6 元
  - 提货方式：[自提] [代运]（二选一，必选）
    ├─ 自提：无运费
    └─ 代运：输入收货地址 → 系统预估运费
  ↓
系统预览：
  - 商品匹配度
  - 预估运费（如代运）
  - 预估到货时间
  ↓
买家确认 → 进入买单池（等待撮合）
```

#### 4.3.2 等待撮合阶段

```
买家订单进买单池
  ↓
AI 撮合引擎扫描（实时）：
  - 商品、等级、数量、价格区间、时间、交付方式
  ↓
3 种匹配结果：
  ① 精确匹配 → 立即成交（自动配对卖家）
  ② 部分匹配 → AI 推荐替代选项（买家手动确认或拒绝）
  ③ 暂无匹配 → 订单持续等待（买家可取消）
```

#### 4.3.3 成交后支付

```
匹配成功 → 系统通知买家："您的订单已成交"
  ↓
显示最终信息：
  - 成交卖家身份：隐藏（只显示"某市郊县农户"）
  - 成交价格：区间交集的中间值
  - 商品详情：卖家上传的 3 张实物图 + 等级描述
  - 交付方式：确认选择（自提 / 代运）
  - 运费：最终额度（如代运）
  ↓
买家付款：
  - 钱进市场企业支付宝账户（平台托管）
  - 收据立即发送
  ↓
系统自动触发：
  - 通知农户："订单已成交，请在 T 时间前送达市场"
  - 通知市场："准备接收该订单"
  - 订单状态 → "已支付，待农户送货"
```

#### 4.3.4 等待收货阶段

```
农户送货 → 市场收货 → 工作人员扫码入库
  ↓
通知买家："您的订单已到市场"
  ↓
买家 App 显示：
  - 订单状态：已入库
  - 货物照片：市场拍摄的 3 张照片
  - 货物等级：目视判定的最终等级
  - 建议提货时间：避开高峰时段
```

#### 4.3.5 提货 / 代运阶段

**路径 A：自提**
```
买家/司机到市场
  ↓
向工作人员报订单号或提货码（4 位）
  ↓
工作人员 App 核验 → 显示货物位置
  ↓
买家清点货物 → 签字确认
  ↓
系统更新：订单状态 → "已提货" → 7 天异议期开始
```

**路径 B：代运**
```
系统自动调用货拉拉 API
  ↓
货拉拉司机接单 → 到市场取货
  ↓
工作人员协助装车 → 拍照存证
  ↓
货拉拉物流跟踪送达 → 买家签收
  ↓
系统更新：订单状态 → "已送达" → 7 天异议期开始
```

#### 4.3.6 异议期 & 结算

```
7 天异议期（从提货/送达之日开始）
  ↓
┌─ 无异议 ─────────────────┐
│ 7 天后自动确认收货         │
│ 订单状态 → "已完成"        │
│ 本周一自动从支付宝扣款      │
└─────────────────────────┘
│
└─ 有异议 ──────────────────┐
  │ 点"申请退货/赔偿"        │
  ↓                        │
  代理人仲裁                 │
  ↓                        │
  ├─ 退款 → 原路返回         │
  ├─ 部分赔付 → 自动扣款     │
  └─ 驳回 → 订单完成         │
                           │
                    支付宝自动结算
```

#### 4.3.7 买家取消流程（对称规则）

```
买家在支付后可取消（不同时段扣款不同）

> T - 24h：扣 1% 服务费
T - 24h ~ T - 5h：扣 5% 服务费
T - 5h ~ T：扣 20% 服务费
T 时间后（农户已送达）：禁止取消，进入"代销流程"
  → 货进"紧急代销池"
  → 24h 内 AI 撮合其他买家
  → 未售完 → 每 6h 降价 5%
  → 48h 未售 → 平台按底价收购
  → 买家承担 30% 违约金

不可抗力：100% 退款（凭证明）
```

### 5.3 模块六：简易等级验收（替换原"标准化等级体系"）

#### 5.3.1 第一期简化策略

v4.0 的"标准化等级体系"按品类详细定义等级标准，第一期过于复杂。v4.1 简化为：

```
每个品类只分 3 档：
  - 特级（精品）
  - 一级（常规）
  - 二级（瑕疵可接受）

挂单时必选一档，由农户自报
市场收货时由工作人员目视 + 拍照判定
```

#### 5.3.2 市场收货判定的三种结果

```
① 符合等级 → 正常入库
② 需要降级（如一级 → 二级）→ 通知买家确认差价
③ 严重不符 / 腐烂 / 数量严重不足 → 拒收 → 触发农户违约流程
```

#### 5.3.3 等级争议

买家收货后 7 天异议期内对等级有异议 → 代理人介入仲裁（见模块九）。

### 5.7 模块十：代理人系统（扩展）

#### 5.7.1 代理人定位（v4.1）
- 平台自有团队，**第一期至少 2-3 人**（比 v4.0 的 2 人略多，保证紧急呼叫的接通率）
- 靠提成 + 仲裁补贴 + 紧急调货补贴激励
- 新增职责：**接收农户紧急呼叫 + 紧急调货**（见第 X 章）

#### 5.7.2 代理人收入构成（v4.1 扩展）

| 收入类型 | 计算方式 | 说明 |
|---------|---------|------|
| 撮合提成 | 服务费 × 20% | 代理人介入撮合的订单 |
| 仲裁补贴 | 10 元/单 | 仲裁完成的争议订单 |
| **紧急订单提成** ⭐ | 服务费 × 50% | 紧急调货成功的订单 |
| **紧急调货补贴** ⭐ | 30 元/单 | 每笔调货成功补贴 |

#### 5.7.3 代理人电话策略（v4.1 新增）

```
代理人电话直接公开真实号码（不使用隐私号）
  - 显示在农户 App 中
  - 农户用自己手机直接拨打
  - 通话无中转

理由：
  ✅ 节省成本、提高接通率
  ✅ 简化技术对接
  ✅ 符合农户使用习惯
风险与应对：
  - 代理人离职时统一更换号码
  - 入职签署"工作号码使用授权"
```

## 六、模块 X：农户取消与违约兜底机制 ⭐（v4.1 新增）

### X.1 设计原则

| 原则 | 说明 |
|---|---|
| **农户有退路** | 农业生产有天气、产量等不可控因素 |
| **买家有保障** | 买家已付款，必须保证订单履约或全额退款 |
| **违约有成本** | 保证金 + 信用分双重惩罚 |
| **平台有兜底** | 代理人 + AI + 自动取消三段兜底 |
| **规则可预期** | 所有惩罚前置告知 |

### X.2 农户保证金（见 5.10.2）

### X.3 取消时间窗与惩罚阶梯

#### X.3.1 时间基准

```
T = 农户在挂单时承诺的"送达市场时间"
所有取消判定以 T 为基准
```

#### X.3.2 惩罚阶梯

| 取消时机 | 信用扣分 | 保证金扣除 | 兜底动作 |
|---|---|---|---|
| **> T - 24h** | -10 | 5% | 自动重新撮合 |
| **T - 24h ~ T - 5h** | -20 | 15% | 代理人介入调货 |
| **T - 5h ~ T - 1h** | -30 | 30% | 代理人紧急调货 |
| **< T - 1h** | 禁止主动取消 | - | 只能走"协商解除" |
| **超过 T 未送达（失联）** | -50 | 100% | 强制兜底 + 黑名单 |
| **不可抗力** | 0 | 0 | 顺延或退款 |

#### X.3.3 不可抗力凭证
- 当地气象局红色/橙色预警截图
- 县级以上医院诊断书
- 交通事故认定书 + 报警回执
- 政府封路通告

### X.4 紧急调货流程（并行三段兜底）

#### X.4.1 整体架构

```
农户申请取消
  ↓
[瞬间] 通知买家，买家三选一：
  ① 必须按原时间到货（默认）
  ② 可接受顺延至次日 → AI 扩大搜索 + 等待时长 +1 小时
  ③ 直接退款
  ↓
[并行启动] 第一段 + 第二段
  ↓
┌─────────────────────────────┐  ┌─────────────────────────────┐
│ 第一段：农户主动呼叫代理人    │  │ 第二段：AI 自动调货（同时）   │
│ - 系统提供真实电话列表       │  │ - 扫描挂单池                │
│ - 30 分钟硬性时长            │  │ - 推送前 5 名备选农户        │
│ - 代理人后台自动弹窗确认      │  │ - 动态等待时长：             │
│ - 需满足"有效呼叫"条件        │  │   R>12h: 2h / 2-12h: 1h     │
│   才能解锁"转 AI"            │  │   <2h: 30min / <30min: 15min│
└─────────────────────────────┘  └─────────────────────────────┘
  ↓ 任一段成功 → 订单完成
  ↓ 两段都失败
第三段：自动取消订单
  ↓
全额退款 + 5% 赔付（从农户保证金扣）
农户信用 -50 + 保证金扣 100%
```

#### X.4.2 "有效呼叫"双条件（满足任一即可转 AI）

- **条件 A**：至少 1 通与代理人的真实通话 > 0 秒
- **条件 B**：已拨打 ≥ 3 个不同代理人，每通响铃 ≥ 10 秒均未接通

#### X.4.3 代理人后台自动确认

```
农户拨打 → 代理人手机响铃 → 代理人后台自动弹出悬浮窗：
┌──────────────────────────────┐
│ 📞 农户张三正在呼叫您          │
│ 订单 #XXX 紧急调货             │
│ 倒计时 3 秒后自动确认接通...    │
│ [立即确认] [这不是我接的]       │
└──────────────────────────────┘
3 秒后自动确认 → 解锁农户端"转 AI"按钮
```

#### X.4.4 并行调货的冲突处理

- 代理人先找到 → AI 任务中止，已推送的备选农户收到安抚消息
- AI 先找到 → 暂存为"备选方案"，等第一段确认
- 两边都找到 → 优先采用代理人的（经验值更高）

#### X.4.5 AI 全自动签约边界

```
✅ 订单金额 ≤ 10,000 元
✅ 价格、等级、送达时间完全一致
❌ 超过 10,000 元 → 必须人工确认
❌ 任何参数变动 → 必须双方确认
```

#### X.4.6 买家偏好的三种选择对 AI 调货策略的影响

| 买家选择 | AI 搜索范围 | AI 等待时长调整 |
|---|---|---|
| ① 必须按原时间 | 只推送"能在 T 前送达"的农户 | 不变 |
| ② 可接受顺延次日 | 扩大范围（允许次日送达） | +1 小时 |
| ③ 直接退款 | 跳过 AI，立即进入第三段 | - |

### X.5 替代货差价分摊

#### X.5.1 替代货比原价贵
```
1. 优先从违约农户保证金扣除
2. 不足部分平台垫付 → 计入农户"待追偿账户"
3. 农户后续订单分账时每笔扣 30% 偿还
4. 买家不承担差价
```

#### X.5.2 替代货比原价便宜
```
100% 差价退买家（作为延误补偿）
```

#### X.5.3 等级不一致
- 降级：必须买家同意 + 按降级价计算 + 违约农户额外 -10 分
- 升级：不需买家同意 + 价格不变 + 替代农户额外补贴 20 元

#### X.5.4 大幅偏离的强制确认
任一项超过以下阈值必须买家确认（30 分钟内回复）：
- 价格偏离 > 15%
- 等级降低
- 送达时间延迟 > 4 小时
- 数量短缺 > 5%

### X.6 买家对称取消规则

| 取消时机 | 信用扣分 | 退款比例 |
|---|---|---|
| > T - 24h | -5 | 99%（扣 1%） |
| T - 24h ~ T - 5h | -15 | 95%（扣 5%） |
| T - 5h ~ T | -30 | 80%（扣 20%） |
| 农户已送达后 | 禁止主动取消 | 进入"代销流程" |
| 不可抗力 | 0 | 100% |

**已送达后拒收处理**：
- 货物进入"紧急代销池"
- 24h 内 AI 撮合其他买家，按实际价结算
- 24h 未售 → 每 6h 降价 5%
- 48h 未售 → 平台按底价收购，记损耗
- 原买家承担 30% 违约金 + 信用 -30

### X.7 风险告知

农户首次挂单、买家首次下单前必须强制弹窗阅读违约规则并勾选同意。

### X.8 数据埋点
- 农户违约率 / 紧急调货成功率 / 平均兜底耗时 / 平台垫付总额 / 不可抗力申诉率 / 买家取消率

## 七、模块 Y：市场运营模块 ⭐（v4.1 新增，第一期简化版）

### Y1 市场角色与资金流

#### Y1.1 市场在流程中的定位

```
农户 ──(挂单)──> 平台
                 ↓ AI 撮合
买家 ──(挂单)──> 平台
                 ↓ 成交 + 买家付款
农户 ──(送货)──> 市场（实体场所）
                 ↓ 收货确认
买家 ──(提货 or 代运)──> 收货
                 ↓
              每周一结算农户
```

#### Y1.2 第一期资金流（最简版）

```
买家付款 → 市场企业支付宝账户（托管）
           ↓
     收货 + 买家已签收 + 7 天异议期结束
           ↓
     平台分账：
       - 货款挂"待结算农户款"
       - 服务费 → 市场运营账户
       - 运费 → 运输方（市场代付货拉拉）
           ↓
     每周一统一结算给农户
```

**简化说明**：
- 第一期不做即时自动分账，每周一手动触发
- 发票第一期线下手工开，不进产品
- 法律定性按"市场代收代付"走，后期再合规改造

### Y2 农户送货流程

#### Y2.1 整体流程

```
撮合成功 → 系统通知农户："请在 [T 时间] 前送达市场"
          ↓
农户 App 显示：
  - 订单详情
  - 市场地址 + 导航
  - 市场联系电话
  - 建议送达时段（T 前 1-4 小时）
          ↓
农户按时送到市场
          ↓
工作人员扫订单二维码 → 开始收货
```

#### Y2.2 时间窗约束

- **市场开放时间**：6:00 - 20:00（可在管理后台配置）
- **建议窗口**：T 时间往前推 1-4 小时
- **超时处理**：超市场开放时间 → 订单顺延到次日开放时间

#### Y2.3 农户端 App 界面

```
┌────────────────────────────┐
│ 订单 #XXX                   │
│ 花生 2000 斤 × 3.45 元       │
│ 送达前：4 小时 30 分钟        │
│                            │
│ 📍 送到：XX 蔬菜市场          │
│ 📞 市场电话：0536-XXX XXXX   │
│                            │
│ [开始送货] [申请取消]         │
└────────────────────────────┘
```

### Y3 市场收货与简易分级

#### Y3.1 工作人员角色（合并）

第一期只有一个角色：**市场工作人员**。
一人兼任收货、分级、出货、代运装车。第二期业务量起来再拆分。

#### Y3.2 工作人员 App 简易后台

```
【今日待处理】

待收货：
  订单 #001 花生 2000 斤 农户：张三 09:00 [开始收货]
  订单 #002 白菜 500 斤 农户：李四 10:00 [开始收货]

待出货：
  订单 #001 花生 2000 斤 买家：王超市 提货码 A2B7 [核对并放行]
```

#### Y3.3 收货流程

```
扫订单二维码 → 过磅 → 输入实际重量
  ↓
对照农户自报等级，目视判断：
  [✓ 符合]  [↓ 降级]  [✗ 拒收]
  ↓
拍 3 张照片上传
  ↓
点"确认入库"
  ↓
系统自动更新订单状态 + 通知买家
```

#### Y3.4 三种收货结果

**情况 1：符合等级** → 直接入库

**情况 2：降级**
```
标记"降级一档" → 系统通知买家：
  "货物等级由[一级]降为[二级]，差价 XX 元将退回"
  [接受] → 差价自动从农户货款扣除退买家
  [不接受] → 进入拒收流程
```

**情况 3：拒收**（严重不符 / 腐烂 / 数量严重不足）
```
标记"拒收" + 拍照存证
  ↓
触发农户违约流程（见第 X 章）
  ↓
货物由农户自行运回 或 按废品处理
```

#### Y3.5 重量不符处理

```
短缺 < 5%  → 按实际重量结算，差额退买家
短缺 5-15% → 必须通知买家确认
短缺 > 15% → 按"拒收"处理
```

#### Y3.6 暂存（简化）

- 第一期不做精细暂存位管理
- 免费暂存：订单完成前免费
- 超时暂存：**24 小时后每天 50 元**（不足 1 天按 1 天算），从买家账户扣

### Y4 买家提货与代运

#### Y4.1 两条路径

买家挂单时必选：
- **自提**：无运费
- **代运**：填地址，平台预估运费，付款时一起付

#### Y4.2 路径 A：自提

```
货物入库 → 系统生成 4 位提货码 → 推送给买家
  ↓
买家/司机到市场 → 向工作人员报订单号或提货码
  ↓
工作人员 App 输入提货码 → 系统显示订单详情
  ↓
买家清点货物 → 签字确认
  ↓
工作人员点"放行" → 出场
  ↓
订单状态 → "已提货" → 7 天异议期开始
```

**第一期只验提货码**，不验身份证。

#### Y4.3 路径 B：代运（只接货拉拉）

```
货物入库 → 工作人员打开"代运工作台" → 一键呼叫货拉拉
  ↓
系统调用货拉拉 API 下单（取货地、送货地、货物信息）
  ↓
司机接单 → 到市场取货 → 工作人员装车 + 拍照
  ↓
货拉拉 App 物流跟踪
  ↓
送达 → 买家签收
  ↓
订单状态 → "已送达" → 7 天异议期开始
```

#### Y4.4 运费结算

- 买家下单时系统预估 → 随货款一起付
- 实际运费由市场直接付货拉拉
- 差额处理：
  - 实际 < 预估：差额退买家
  - 实际 > 预估 10% 以内：市场承担
  - 实际 > 预估 10%：工作人员联系买家协商

#### Y4.5 暂不做
- 拼车、多物流商对比、冷链车（第二期）

### Y5 农户结算

#### Y5.1 结算规则

```
结算周期：每周一结算上周所有"已完成"订单
完成定义：买家已过 7 天异议期（或提前确认收货）

结算金额 = 货款总额 - 差价扣款 - 损耗扣款 - 保证金扣款

结算方式：
  第一期：工作人员在管理后台手工触发
  操作：选中农户本周订单 → 生成结算单 → 支付宝/微信/银行转账
```

#### Y5.2 农户结算后台

  订单 #001 花生 2000 斤 × 3.45 = 6900 ✓
  订单 #003 花生 1500 斤 × 3.40 = 5100 ✓
  订单 #007 拒收              = 0 ✗

应付货款：12000 元
扣除：
  - 订单 #001 降级差价：-200
  - 订单 #007 保证金扣除：-300
实付：11500 元

[生成结算单] [标记已支付]

#### Y5.3 农户端查看

农户 App 可查看本周/本月订单、收入明细、扣款明细、历史结算。

#### Y5.4 提前支付

支持**人工预支申请**：农户 App 申请 → 工作人员审核 → 手动转账。每周限 1 次。

### Y6 损耗责任

| 阶段 | 损耗类型 | 责任方 |
|---|---|---|

| 提货后 7 天内 | 收货争议 | **代理人仲裁** |

### Y7 管理后台必要页面

### 9.4 关键 API 对接清单（v4.1 新增）

#### 9.4.1 支付宝企业账户对接

**场景**：买家支付 → 资金托管 → 市场每周一结算给农户

```
接口：支付宝"资金托管" + "交易组件" + "分账"
文档：https://open.alipay.com/api/detail/alipay.trade.page.pay

关键流程：
1. 生成支付链接
   POST /alipay/trade/page/pay
   参数：
     - app_id: 市场企业支付宝 app_id
     - subject: "农产品订单 #XXX"
     - total_amount: 货款 + 服务费 + 运费
     - return_url: 支付成功回调地址
     - notify_url: 异步通知地址

2. 支付成功回调
   POST {notify_url}（异步）
   参数：
     - trade_no: 支付宝交易号
     - out_trade_no: 我们的订单号
     - total_amount: 实际支付金额
   ↓
   系统验证 + 订单状态更新为"已支付"

3. 周一自动结算给农户
   (第一期手工触发，第二期改自动)
   - 查询本周已完成订单
   - 汇总金额（扣差价、保证金、损耗后）
   - 调用支付宝转账接口
     POST /alipay/fund/trans/uni/transfer
     参数：
       - out_biz_no: 结算单号
       - amount: 应付农户金额
       - payee_account: 农户账户（支付宝/微信/银行卡）
```

#### 9.4.2 货拉拉 API 对接

**场景**：买家选"代运" → 一键调用货拉拉下单

```
接口：货拉拉企业 API
文档：联系货拉拉商务获取（v3 版本）

关键流程：
1. 查询预估运费（买家挂单时）
   POST /api/carpool/order/estimatedPrice
   参数：
     - startCity: 市场所在城市（寿光）
     - startAddress: 市场具体地址
     - endCity: 买家所在城市
     - endAddress: 买家收货地址
     - cargoWeight: 订单重量
   ↓
   返回 priceRange（预估价格）

2. 创建订单（货物入库后工作人员一键）
   POST /api/carpool/order/createOrder
   参数：
     - startCity / startAddress: 市场信息
     - endCity / endAddress: 买家信息
     - cargoWeight / cargoSize: 货物信息
     - estimatedPrice: 预估价格
     - cargoInfo: 品类（蔬菜、水果等）
     - customerPhone: 买家电话
   ↓
   返回 orderId（货拉拉订单号）

3. 实时物流推送
   货拉拉主动推送：
   - 司机已接单
   - 司机已取货
   - 运输中（推送定位）
   - 已送达
   ↓
   我们订单状态同步更新

4. 结算
   - 实际运费由市场直接支付货拉拉
   - 差额处理（见 Y4.4）
```

#### 9.4.3 阿里云实名认证

**场景**：农户和买家首次注册需实名

```
接口：阿里云"人脸识别 + 身份验证"
文档：https://help.aliyun.com/zh/

关键流程：
1. 初始化认证
   POST /api/realname/verify/init
   参数：
     - idType: "IDCARD"
     - idNumber: 身份证号
     - name: 真实姓名
     - mobile: 手机号
   ↓
   返回认证 URL（跳转到人脸识别页面）

2. 轮询查询结果
   GET /api/realname/verify/query?bizId={bizId}
   返回：
     - status: "PASS" / "FAIL" / "PROCESSING"
     - failReason: 失败原因（可选）
   ↓
   通过后，系统更新用户认证状态
```

#### 9.4.4 阿里云短信服务

**场景**：订单通知、代理人呼叫提醒

```
接口：阿里云短信 API
文档：https://help.aliyun.com/zh/sms/

关键场景：
1. 农户送货通知
   模板：您的订单 #XXX 已成交，请在 {time} 前送达 {market_name}。地址：{address} 电话：{phone}

2. 买家到货通知
   模板：您的订单 #XXX 已到市场，可按以下方式提货...

3. 代理人紧急呼叫提醒
   模板：[紧急]农户 {farmer_name} 订单 {order_id} 需紧急调货，请立即处理

4. 异议期提醒
   模板：您的订单进入 7 天异议期，如有异议请点击链接申诉...

实现：
POST /SendSms
参数：
  - PhoneNumbers: 目标手机号
  - TemplateCode: 对应的短信模板 ID
  - TemplateParam: JSON 格式的动态参数
```

#### 9.4.5 极光推送

关键场景：

## 十四、关键流程图汇总

### 14.1 完整成交流程（v4.1）

```
卖家挂单 → 卖单池 ──┐
                   │
买家挂单 → 买单池 ──┼─→ AI 撮合 ─→ 成交 ─→ 买家付款
（选自提/代运）      │                              │
                   │                              ↓
                   │                    通知农户送货（含 T 时间）
                   │                              │
                   │                              ↓
                   │              ┌───正常送达───┐         ┌──异常──┐
                   │              ↓              │         ↓       │
                   │          市场收货           │      农户违约    │
                   │          过磅+分级+拍照      │         ↓       │
                   │              ↓              │     紧急兜底(第X章)
                   │          ┌───正常          │         ↓       │
                   │          │ 降级→买家确认    │    代理人+AI 调货 │
                   │          │ 拒收→违约流程    │         ↓       │
                   │          └───入库          │     成功/失败    │
                   │              ↓              │         ↓       │
                   │          通知买家可提货      │     成功→继续    │
                   │              ↓              │     失败→退款    │
                   │        ┌──自提──┐ ┌──代运──┐
                   │        │提货码  │ │呼叫货拉拉
                   │        └──────┘ └──────┘
                   │              ↓
                   │          7 天异议期
                   │              ↓
                   │       ┌─无异议→自动确认
                   │       └─有异议→代理人仲裁
                   │              ↓
                   │        每周一结算农户
                   │              ↓
                   │          订单完成
                   ↓
              （全程暗池，双方互不相识）
```
