## 前言：为什么是简化版

第一期目标是"**验证这个商业模式能不能跑通**"，不是做终态产品。所以：
- 法律定性暂不深究，按"市场收款后结算给农户"的直接模式走（后期再做合规改造）
- 复杂的库存/分级/质检/员工分工全部合并简化
- 发票问题第一期手工处理，不进产品
- 只保留 5 个必要模块，其余等第二期

## 模块 Y1：市场角色定义（第一期简化）

### Y1.1 市场在流程中的定位

```
农户 ──(挂单)──> 平台
                  ↓ AI 撮合
                  ↓
买家 ──(挂单)──> 平台
                  ↓ 成交
                  ↓
农户 ──(送货)──> 市场（实体场所，你的主场）
                  ↓ 收货确认
                  ↓
买家 ──(提货 or 代运)──> 收货
                  ↓
                市场结算给农户（按周）
```

### Y1.2 第一期资金流（最简版）

```
买家付款 → 市场的支付宝企业账户（托管）
           ↓
     市场收到货、买家收到货
           ↓
     平台自动分账：
       - 货款 → 挂在"待结算农户款"（先不给农户）
       - 服务费 → 市场运营账户
       - 运费 → 运输方
           ↓
     每周一统一结算给农户（支付宝/微信/银行转账均可）
```

**为什么不即时结算给农户？**
- 避免高频小额转账的手续费
- 给市场留出"损耗扣款、质量争议"的缓冲时间
- 简化财务对账工作

**发票问题第一期处理方式**：
- 大客户（超市、餐饮连锁）需要发票 → 市场线下手工开增值税普通发票
- 小客户不要发票 → 不开
- 这部分等第二期做成系统化

## 模块 Y2：农户送货流程（合并 N2+N4）

### Y2.1 整体流程

```
撮合成功，订单生成
        ↓
系统通知农户："您的订单已成交，请在 [T 时间] 前送达市场"
        ↓
农户 App 显示：
  - 订单详情
  - 市场地址 + 导航
  - 市场联系电话
  - 建议送达时段（如 8:00-10:00 避开高峰）
        ↓
农户按时送到市场门口
        ↓
工作人员在 App 扫订单二维码 → 开始收货
        ↓
过磅 → 目视检查 → 拍 3 张照片 → 确认入库
        ↓
系统自动：
  - 更新订单状态为"已入库"
  - 通知买家"货已到市场，可安排提货"
```

### Y2.2 送货时间窗（简化版）

第一期不做精细的预约时段，只做两件事：

1. **系统给农户一个建议时间窗**
   - 基于订单 T 时间，往前推 1-4 小时
   - 例如 T = 10:00，建议窗口 "6:00 - 10:00 送达"

2. **市场开放时间硬性约束**
   - 市场开放时间：6:00 - 20:00（可在管理后台配置）
   - 农户只能在这段时间送达
   - 超出时间 → 订单顺延到次日开放时间

**不做**（第二期再考虑）：
- 时段预约（防止扎堆）
- 车位安排
- 入场排队

### Y2.3 农户端 App 界面简化

```
我的订单 → [进行中]
┌────────────────────────────┐
│ 订单 #XXX                   │
│ 花生 2000 斤 × 3.45 元       │
│ 送达前：4 小时 30 分钟        │

│ 📍 送到：XX 蔬菜市场          │
│    山东寿光 XX 路 XX 号       │
│    [导航]                   │
│                            │

│                            │
│ [开始送货] [申请取消]         │

```

## 模块 Y3：市场收货与简易分级（合并 N3+N6）

### Y3.1 工作人员角色（第一期只有一个）

第一期不分"收货员/质检员/出货员/结算员"，**只有一个角色：市场工作人员**

一个人负责：
- 收货入库
- 等级核对
- 买家提货放行
- 代运装车

等第二期业务量起来再拆分。

### Y3.2 工作人员 App（简易后台）

```
【今日待处理】

┌─ 待收货 ─────────────┐
│ 订单 #001 花生 2000 斤│
│ 农户：张三 预计 09:00 │
│ [开始收货]            │
├──────────────────────┤
│ 订单 #002 白菜 500 斤 │
│ 农户：李四 预计 10:00 │
│ [开始收货]            │
└──────────────────────┘

┌─ 待出货 ─────────────┐
│ 订单 #001 花生 2000 斤│
│ 买家：王超市          │
│ 预计提货：11:00       │
│ 提货码：A2B7          │
│ [核对并放行]          │
└──────────────────────┘
```

### Y3.3 简易收货流程

```
工作人员点"开始收货"
    ↓
扫农户的订单二维码（验证订单真实性）
    ↓
过磅 → 输入实际重量
    ↓
对照农户自报等级 → 目视判断：
  [✓ 符合等级]  [↓ 降级]  [✗ 拒收]
    ↓
拍 3 张照片上传（作为证据）
    ↓
填写简短备注（可选）
    ↓
点"确认入库"
    ↓
系统自动：
  - 订单状态 → "已入库"
  - 通知买家"货已到"
  - 通知农户"已收货"
```

### Y3.4 三种收货结果的处理

#### 情况 1：符合等级（正常）
```
直接入库 → 订单正常进行
```

#### 情况 2：需要降级（如一级→二级）
```
工作人员标记"降级一档"
    ↓
系统立即通知买家：
  "您的订单货物等级由[一级]降为[二级]，
   差价 XX 元将退回您的账户，
   是否接受？"
    ↓
[接受] → 订单继续，差价自动从农户货款扣除退买家
[不接受] → 进入"拒收流程"
```

#### 情况 3：拒收（严重不符 / 腐烂 / 数量严重不足）
```
工作人员标记"拒收" + 拍照存证
    ↓
系统自动触发"农户违约流程"（走 X.4 章节的紧急调货）
    ↓
农户处罚：信用 -30，保证金扣 30%
    ↓
货物由农户自行运回 或 按废品处理（农户选择）
```

### Y3.5 实际重量与订单不符

```
订单 2000 斤，实际过磅 1950 斤（短缺 50 斤）

规则：
  短缺 < 5% → 按实际重量结算（差额退买家）
  短缺 5-15% → 通知买家是否接受
  短缺 > 15% → 按"拒收"处理
```

### Y3.6 暂存位简化处理

第一期**不做精细暂存位管理**：
- 市场有多大暂存区，工作人员心里有数
- 订单入库后直接在现场堆放，用小白板或小牌子标订单号
- 免费暂存：订单完成前免费
- 超时暂存（买家拖着不来）：24 小时后每天加收 50 元暂存费

第二期再做"暂存位编号 + 电子标签 + 自动定位"。

## 模块 Y4：买家提货与代运（合并 N7）

### Y4.1 两条路径

买家在挂单时就要选：

```
买家挂单 → AI 对话引导：
  "您是自己来提货，还是让我们帮您送？"
    ↓
  [我自己提货]  → 不收运费，提货码核验
  [帮我代运]    → 显示预估运费，付款时一起付
```

### Y4.2 路径 A：买家自提

#### Y4.2.1 提货码生成
```
货物入库 → 系统生成 4 位提货码
    ↓
推送给买家：
  "您的订单 #XXX 已到市场，
   提货码：A2B7
   请在 [时间] 内到市场提货
   地址：XX 蔬菜市场 XX 档口"
```

#### Y4.2.2 现场核验流程
```
买家（或司机）到达市场
    ↓
到工作人员处报订单号或提货码
    ↓
工作人员在 App 输入提货码
    ↓
系统显示订单详情 + 货物存放位置
    ↓
工作人员指引买家找到货物
    ↓
买家清点货物 → 签字确认
    ↓
工作人员点"放行" → 出场
    ↓
系统自动：
  - 订单状态 → "已提货"
  - 7 天异议期开始
```

#### Y4.2.3 身份核验（简化）
第一期**只验提货码**，不验身份证。
- 提货码泄露风险：由买家自己负责
- 第二期再加"买家授权司机"功能

### Y4.3 路径 B：市场代运

#### Y4.3.1 简化代运
第一期**只对接货拉拉一家**（最容易接入）：

```
货物入库 → 系统通知工作人员"有代运订单"
    ↓
工作人员打开"代运工作台"：
  [订单 #XXX 货物已就位，需要发货]
  [一键呼叫货拉拉]
    ↓
系统调用货拉拉 API 下单
  - 取货地：本市场
  - 送货地：买家地址
  - 货物信息：品类 + 重量
    ↓
货拉拉司机接单 → 到市场取货
    ↓
工作人员协助装车 → 拍照存证
    ↓
货拉拉 App 跟踪物流
    ↓
送达 → 买家签收
    ↓
系统更新订单状态 → "已送达"
```

#### Y4.3.2 运费
- 下单时系统调用货拉拉预估运费 → 显示给买家
- 买家付款时一起付（进市场托管账户）
- 实际运费由市场直接付给货拉拉
- 差额处理：
  - 实际 < 预估 → 差额退买家
  - 实际 > 预估（超 10%）→ 市场承担（运营成本），不找买家加钱
  - 实际 > 预估（超 30%）→ 工作人员联系买家协商

#### Y4.3.3 暂不做
- 拼车（第二期）
- 多家物流对比（第二期）
- 冷链车（需要专门对接，第二期）

## 模块 Y5：农户结算（简化）

### Y5.1 结算规则

```
结算周期：每周一结算上周所有"已完成"订单
结算金额 = 货款总额 - 差价扣款 - 损耗扣款 - 保证金补缴

完成定义：买家已过 7 天异议期（或提前确认收货）

结算方式：
  第一期：工作人员在管理后台手工触发结算
  操作：选中本周农户订单 → 点"生成结算单" → 确认 → 支付宝/微信转账
```

### Y5.2 农户结算后台页面

```
【本周待结算】

农户：张三（138-XXXX-1234）
本周订单：
  订单 #001 花生 2000 斤 × 3.45 元 = 6900 元 ✓
  订单 #003 花生 1500 斤 × 3.40 元 = 5100 元 ✓
  订单 #007 拒收              = 0 元 ✗

应付货款：12000 元
扣除：
  - 订单 #001 降级差价：-200 元
  - 订单 #007 保证金扣除：-300 元
实付：11500 元

[生成结算单] [标记已支付]
```

### Y5.3 农户端查看结算

农户 App 可以查看：
- 本周/本月订单列表
- 每笔订单的收入
- 扣款明细
- 历史结算记录
- 下次结算日

### Y5.4 提前支付（可选）

第一期不做自动预支，但支持**人工预支申请**：
- 农户 App 点"申请提前结算" → 工作人员审核 → 手动转账
- 适用于农户资金周转困难的情况
- 每周限 1 次

## 模块 Y6：损耗与场地费（简化）

### Y6.1 损耗责任（简化版）

| 阶段 | 损耗类型 | 责任方 |
|---|---|---|
| 送货到市场前 | 车上颠坏、运输中变质 | **农户** |
| 市场收货时发现不合格 | 拒收 | **农户** |
| 市场暂存期间（≤ 24h） | 正常损耗 | **市场** 承担 |
| 市场暂存期间（> 24h，因买家拖延） | 超时损耗 | **买家** 承担（从暂存费出） |
| 代运途中损坏 | 运输损坏 | **运输方**（货拉拉） |
| 买家提货后 7 天内发现问题 | 收货争议 | **代理人仲裁** |

### Y6.2 场地费（第一期先不做）

第一期平台收入只有"服务费 1%"这一项，场地费暂不纳入产品：
- 你的线下市场本来就有场地费收入
- 这部分继续走你原来的线下收费方式
- 不和平台订单挂钩

**等到第二期**，如果要做"场地费线上化"，可以考虑：
- 按月/季度收取固定摊位费
- 和农户保证金绑定（场地费欠缴 → 限制挂单）
- 这是独立于订单的收入
