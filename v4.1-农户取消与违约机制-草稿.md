## 模块 X：农户取消与违约兜底机制 ⭐

### X.1 设计原则

| 原则 | 说明 |
|---|---|
| **农户有退路** | 农业生产存在天气、产量等不可控因素，必须留出取消通道 |
| **买家有保障** | 买家已付款，平台必须保证订单最终履约或全额退款 |
| **违约有成本** | 通过保证金 + 信用分双重惩罚，遏制随意违约 |
| **平台有兜底** | 代理人 + AI + 电话三层兜底，保证 7×24 不掉单 |
| **规则可预期** | 所有惩罚阶梯前置告知，挂单前必须签署同意 |

---

### X.2 农户保证金体系

#### X.2.1 保证金与挂单额度

```
新农户实名认证完成 → 默认保证金 0 元 → 不可挂单
农户充值保证金 → 解锁挂单额度

充值金额      可挂单总金额（杠杆 10 倍）
100 元    →   1,000 元
500 元    →   5,000 元
1,000 元  →   10,000 元
3,000 元  →   30,000 元
10,000 元 →   100,000 元

最低充值：100 元（降低门槛，方便小农户试用）
最高额度：100,000 元（一期）
```

#### X.2.2 保证金冻结规则

```
农户挂单时：
  按订单金额 10% 冻结保证金

示例：
  农户挂单 5,000 元的花生
  → 冻结 500 元保证金
  → 剩余可用保证金 = 总保证金 - 已冻结 - 违约扣除

订单完成时：
  解冻 500 元 → 农户可继续挂单或提现
```

#### X.2.3 保证金不足时的处理

```
情况 1：保证金不足以冻结新订单
  → 禁止挂单
  → 提示农户充值

情况 2：违约扣款后保证金归零
  → 自动冻结所有进行中订单（不允许新挂单）
  → 农户必须补缴保证金才能继续运营
  → 30 天未补缴 → 自动注销账号
```

#### X.2.4 保证金提现

```
提现申请 → T+7 到账（资金沉淀期，防欺诈）
提现期间不可挂单（防止提现后违约）
单笔提现手续费 2 元（覆盖银行通道费）
```

### X.3 取消时间窗与惩罚阶梯

#### X.3.1 时间基准

```
T = 农户在挂单时承诺的"送达市场时间"
取消时机的判定全部以 T 为基准，而非买家取货时间
```

#### X.3.2 农户取消惩罚阶梯

| 取消时机 | 信用扣分 | 保证金扣除 | 兜底动作 | 备注 |
|---|---|---|---|---|
| **> T - 24h** | -10 | 5% | 自动重新撮合 | 时间充裕，影响小 |
| **T - 24h ~ T - 5h** | -20 | 15% | 代理人介入调货 | 推荐预留窗口 |
| **T - 5h ~ T - 1h** | -30 | 30% | 代理人紧急调货 + 顺延评估 | 高风险时段 |
| **< T - 1h** | 禁止主动取消 | - | 只能走"协商解除" | 强制顺延或退款 |
| **超过 T 未送达（失联）** | -50 | 100% | 强制兜底 + 黑名单评估 | 严重违约 |
| **不可抗力** | 0 | 0 | 顺延或退款 | 凭证明，代理人审核 |

#### X.3.3 不可抗力凭证范围

代理人审核以下凭证可免除处罚：

- 当地气象局红色/橙色预警截图（暴雨、冰雹、暴雪等）
- 县级以上医院 24 小时内开具的病历或诊断书
- 交通事故认定书或现场照片 + 报警回执
- 政府发布的疫情管控、道路封闭通告

**审核要点**：凭证时间必须在违约前后 24 小时内，且与事件直接相关。

#### X.3.4 信用分体系

```
新农户初始信用：100 分

封号阈值：
  60 分以下 → 限挂单（每日最多 1 单）
  40 分以下 → 暂停挂单 14 天
  20 分以下 → 封号 7 天
  0 分以下 → 永久封号

信用恢复：
  每完成 1 单无违约订单 → +2 分
  封号期满后 → 信用重置为 50 分（一次机会）
  二次封号 → 永久封号，保证金扣除 50% 作为信用赔偿
```

### X.4 紧急调货流程（农户主动模式）

#### X.4.1 整体流程

本平台采用"**农户主动呼叫 + AI 兜底 + 自动取消**"的三段式机制。
不同于传统的系统自动派单模式，本流程更符合菜农习惯——遇到问题先打电话找人。

```
农户决定取消订单
        ↓
[关键] 系统瞬间通知买家："农户申请取消，平台正在调货"
        ↓
App 弹出"违约处理向导"
        ↓
┌─────────────────────────────┐
│  第一段：农户主动呼叫代理人   │
│  系统提供代理人真实电话列表    │
│  农户依次拨打                 │
│  硬性时长：30 分钟             │
│  超时强制转 AI                 │
│  解锁"转 AI"前提：             │
│    必须真实通话 > 0 秒        │
└─────────────────────────────┘
        ↓ 30 分钟超时 / 农户主动转 AI
┌─────────────────────────────┐
│  第二段：转 AI 自动调节       │
│  AI 扫描挂单池                │
│  推送备选农户                 │
│  等待时长动态调整（见 X.4.3） │
│    剩余 > 12h → 等 2 小时     │
│    剩余 2-12h → 等 1 小时     │
│    剩余 < 2h → 等 30 分钟     │
└─────────────────────────────┘
        ↓ AI 等待超时无响应
┌─────────────────────────────┐
│  第三段：自动取消订单         │
│  全额退款 + 平台赔付 5%       │
│  违约农户扣保证金 + 信用      │
└─────────────────────────────┘
```

**关键设计点**：
- 买家在第一时间被告知，不再被动等待
- 代理人电话公开，不使用隐私号（节省成本，提升直接性）
- AI 等待时长根据剩余时间智能调整（避免一刀切 30 分钟）
- 防止农户跳过呼叫环节"假装"找过代理人

#### X.4.2 第一段：农户主动呼叫代理人

```
触发条件：
  - 农户主动申请取消订单
  - 农户超过 T - 1h 仍未确认送货（系统主动引导）

[同步动作] 系统在该瞬间通知买家：
  推送 + App 内消息 + 短信
  内容："您的订单 #XXX 农户申请取消，
        平台正在紧急调货，预计 30-150 分钟内有结果。
        请选择您的偏好：
        ① 必须按原时间到货（默认）
        ② 可接受顺延至次日
        ③ 直接退款，不要货了"

买家偏好会直接影响 AI 调货策略：
  ① 必须按原时间 → AI 只推送"能在 T 前送达"的农户
  ② 可接受顺延 → AI 扩大搜索范围（次日也可）
                 同时延长 AI 等待时长 +1 小时
  ③ 直接退款 → 跳过第二段，立即进入第三段（退款 + 处罚农户）

买家 5 分钟内未选 → 默认 ①
买家随时可改变选择（在 AI 等待期间）

农户 App 界面：
  ┌──────────────────────────────────┐
  │ ⚠️ 您即将取消订单 #XXX            │
  │                                  │
  │ 原订单：花生 2000 斤 一级 3.45元  │
  │ 送达时间：2026-04-15 10:00        │
  │ 剩余：4 小时 30 分钟               │
  │                                  │
  │ 取消将会：                        │
  │   - 扣信用 -30                    │
  │   - 扣保证金 30%                  │
  │                                  │
  │ 建议先联系代理人协助调货          │
  │ 代理人可能帮您找到替代方案        │
  │                                  │
  │ ⏱️ 第一段倒计时：29:48             │
  │ （30 分钟内必须完成呼叫尝试）     │
  │                                  │
  │ 代理人电话：                      │
  │   📞 张代理 138-XXXX-1234 [呼叫]  │
  │      └ 状态：未拨打                │
  │   📞 李代理 139-XXXX-5678 [呼叫]  │
  │      └ 状态：已拨打 0:32 未接通    │
  │   📞 王代理 137-XXXX-9012 [呼叫]  │
  │      └ 状态：已通话 2:15           │
  │                                  │
  │ [转 AI 处理]（已解锁 ✓）          │
  │ [直接取消订单（接受全部惩罚）]    │
  └──────────────────────────────────┘

电话呼叫机制：
  - 直接显示代理人真实号码（不使用隐私号）
  - 农户使用自己手机直接拨打
  - 系统通过"呼叫日志"识别是否真实通话：
    · App 跳起手机系统拨号 → 拨号后回到 App
    · App 检测呼叫日志（Android 直接读取，iOS 需用户主动确认）
    · 或：系统监测代理人手机是否收到来电（双向核验）

通话识别策略（防作弊）：
  方案 A（推荐）：代理人接听后在代理人后台标记"已接通 - 订单 #XXX"
                这样不依赖手机权限，跨平台通用
  方案 B（备用）：Android 读取通话日志判断
  方案 C（兜底）：用户拒绝授权 → 默认未通话 → "转 AI"按钮不可用

代理人接听后操作：
  - 代理人后台自动提示"农户 XX 来电，订单 #XXX"
  - 代理人点击"接听" → 后台弹出该订单详情
  - 代理人可执行：查找备选农户、强制撮合、调整价格
  - 处理结果回写订单
  - 代理人收益：紧急订单提成 = 服务费 × 50% + 调货成功补贴 30 元

如果代理人无法当场解决（需要时间找货）：
  - 代理人在后台标记"处理中" → 订单暂时冻结
  - 第一段倒计时暂停
  - 处理时限：2 小时
  - 期间农户无需做任何操作

第一段结束的两种方式：
  ① 代理人成功调货 → 流程完结，返回正常履约
  ② 30 分钟到期 / 农户主动点"转 AI"
     → 必须满足"有效呼叫"条件（见下方）
     → 否则按"无故失联"处理（信用 -50，保证金扣 100%）

"有效呼叫"判定规则（满足任一即可解锁"转 AI"）：
  条件 A：至少一通与代理人的真实通话 > 0 秒（代理人接听）
  条件 B：已拨打 ≥ 3 个不同代理人，且每通响铃 ≥ 10 秒均未接通
         （证明农户确实尝试过，不是随便点点）

为什么硬性 30 分钟？
  - 防止农户故意拖延规避惩罚
  - 防止订单卡死无人处理
  - 与买家被告知的"30-60 分钟内有结果"形成预期闭环

为什么要两条"有效呼叫"条件？
  - 条件 A：证明农户真的联系到了人
  - 条件 B：保护农户——如果所有代理人真的都没人接，不应让农户被困
  - 两条合并后既防跳过，又不卡死农户
```

#### X.4.3 第二段：AI 自动调节（与第一段并行）⭐

```
[关键设计] 第二段与第一段并行启动，不等第一段结束

农户申请取消的瞬间：
  - 买家收到通知（含偏好选择）
  - 第一段呼叫代理人开始倒计时
  - [同时] 第二段 AI 调货也开始运转（静默模式）

并行好处：
  - 节省时间，AI 与代理人同时找货
  - 最坏情况下总耗时 = max(第一段, 第二段)，不再是相加
  - 最长 2 小时 → 最长 1 小时（剩余时间充裕时）

冲突处理：
  - 如果代理人先找到货 → AI 任务立即中止，通知已推送的备选农户
  - 如果 AI 先找到货 → 暂存为"备选方案"，等第一段结束后确认
  - 如果两边都找到 → 优先采用代理人找的（代理人通常有经验）

触发条件：
  - 农户申请取消（立即与第一段并行启动）
  - 或代理人接听后处理失败并标记"调货失败"
  - 或农户主动点击"转 AI 处理"（切换到前台显示 AI 进度）

AI 执行步骤：

1. 扫描挂单池
   筛选条件：
   - 商品品类匹配
   - 等级 ≥ 原订单要求
   - 数量充足（含拼单可能）
   - 距离市场 ≤ 200 公里
   - 农户信用 ≥ 70 分
   - 农户当前在线（30 分钟内有 App 活动）

2. 按优先级排序
   优先级 = 距离权重(40%) + 信用权重(30%) + 价格权重(30%)

3. 批量推送（前 5 名备选农户）
   - App 推送 + 短信
   - 内容："紧急调货：花生 2000 斤 一级 3.45元
            4 小时内送达市场，加急补贴 50 元，点击接单"
   - 每个农户独立等待 5 分钟

4. 第一个接单的农户得单
   - 系统自动生成订单
   - 通知原买家"已为您找到备选货源"

AI 全自动签约的边界：
  ✅ 订单金额 ≤ 3,000 元
  ✅ 价格、等级、送达时间完全一致
  ❌ 超过 3,000 元 → 必须人工确认（推送给在线代理人 / 给农户和买家二次确认）
  ❌ 任何参数变动 → 必须双方确认（见 X.5）
```

#### X.4.3.1 AI 等待时长动态调整 ⭐

不同于固定时长，AI 调节的等待时间根据"距离 T 还剩多少时间"动态调整：

```
设 R = T - 当前时间（剩余可用时间）

R > 12 小时 → AI 等待 2 小时
  原因：时间充裕，可慢慢推送，覆盖更多备选农户

2 小时 ≤ R ≤ 12 小时 → AI 等待 1 小时
  原因：中等紧急，平衡推送效果与时间压力

R < 2 小时 → AI 等待 30 分钟
  原因：时间紧迫，不能再等，快速决断

R ≤ 30 分钟 → AI 等待 15 分钟
  原因：极度紧急，几乎不可能调货成功，
        快速失败让买家尽早知情
```

**示例**：
| 农户申请取消时刻 | 距离 T 剩余 | AI 等待时长 | 第二段最晚结束时刻 |
|---|---|---|---|
| T - 24h | 24 小时 | 2 小时 | T - 21.5h |
| T - 6h | 6 小时 | 1 小时 | T - 4.5h |
| T - 1h | 1 小时 | 30 分钟 | T - 0h（卡线） |
| T - 0.5h | 30 分钟 | 15 分钟 | 已超 T |

**调整后的好处**：
- 时间充裕时充分尝试，不浪费机会
- 时间紧迫时快速止损，避免买家干等
- 整体调货成功率预估提升 20-40%

#### X.4.4 第三段：自动取消订单

```
触发条件：
  AI 调节阶段超时（按 X.4.3.1 动态时长）无有效响应

系统自动执行：

1. 标记订单为"调货失败 - 已取消"

2. 退款给买家
   - 全额退款（货款 + 服务费 + 运费）
   - 额外赔付 5% 作为平台违约金（从违约农户保证金扣）
   - 退款 T+1 到账买家支付宝

3. 违约农户处罚
   - 信用扣 -50
   - 保证金扣 100%
     · 优先用于赔付买家 5%
     · 剩余进入平台风险池
   - 进入"严重违约名单"
   - 当月内 3 次严重违约 → 永久封号

4. 通知双方
   - 农户：收到违约处罚通知 + 申诉入口
   - 买家：收到取消通知 + 退款进度

5. 数据归档
   - 订单状态：已取消
   - 取消原因：农户违约 + 调货失败
   - 完整呼叫日志、AI 推送日志、决策时间线全部归档
```

#### X.4.5 流程时间线示例

**示例 1：并行调货 - 代理人先成功（R = 24h）**
```
T = 农户承诺送达市场时间：2026-04-16 10:00

04-15 10:00（T-24h）农户申请取消订单
04-15 10:00 系统瞬间通知买家 → 买家选择 ①"必须按原时间"
04-15 10:00 [并行启动] 第一段呼叫 + 第二段 AI 调货
04-15 10:00 - 10:08 农户拨打张代理通话 1:23
04-15 10:15 张代理后台查到备选农户 C，点"强制撮合"
04-15 10:15 订单完结
           [注]：AI 第二段此时也在跑，收到"代理人已成功"信号后中止推送
```

**示例 2：并行调货 - AI 先成功（R = 6h）**
```
T = 2026-04-15 10:00

04-15 04:00（T-6h）农户申请取消
04-15 04:00 买家选 ①
04-15 04:00 [并行启动] 第一段 + 第二段
04-15 04:00 - 04:30 第一段：农户依次拨打 3 个代理人全部未接通（每通响铃 > 10s）
           满足"有效呼叫"条件 B，"转 AI"按钮解锁
04-15 04:20 AI 已推送备选农户 5 名
04-15 04:25 备选农户 D 接单
04-15 04:25 订单完结
           [注]：第一段被 AI 成功信号打断，不再强制等满 30 分钟
```

**示例 3：全部失败，进入第三段（R = 2h）**
```
T = 2026-04-15 10:00

04-15 08:00（T-2h）农户申请取消
04-15 08:00 买家选 ①
04-15 08:00 [并行启动] 第一段 + 第二段
04-15 08:00 - 08:30 第一段：农户与 2 个代理人通话均未调成
04-15 08:00 - 08:30 第二段：AI 等待 30 分钟（剩余 < 2h），无农户接单
04-15 08:30 第一段超时 + 第二段超时 → 进入第三段
04-15 08:30 系统自动取消订单
04-15 08:30 买家收到全额退款 + 5% 赔付
04-15 08:30 违约农户信用 -50 + 保证金扣 100%
```

**示例 4：买家选"接受顺延"**
```
T = 2026-04-15 10:00

04-15 08:00 农户申请取消
04-15 08:00 买家选 ②"可接受顺延至次日"
04-15 08:00 AI 扩大搜索范围（允许次日送达）+ 等待时长 +1 小时 = 1.5 小时
04-15 08:00 - 09:30 AI 调货
04-15 09:15 备选农户 E 接单，承诺次日 10:00 送达
04-15 09:15 平台补贴买家 10% 货款作为延误补偿
04-15 09:15 订单延期到 04-16 10:00，正常履约
```

**示例 5：农户企图跳过呼叫环节**
```
04-15 08:00 农户申请取消，未拨打任何代理人
04-15 08:00 农户立即点击"转 AI 处理"
           系统拒绝："请至少联系一位代理人"，按钮置灰
04-15 08:30 农户一直没打电话，第一段 30 分钟超时
           → 不满足"有效呼叫"条件（既无通话，也未达 3 通响铃）
           → 强制认定"无故失联"
04-15 08:30 信用 -50，保证金扣 100%
           第二段继续尝试调货（买家订单保留）
           [注]：虽然农户被惩罚，但买家订单仍有救
```

#### X.4.6 代理人电话号码策略

```
本平台采用"明码电话"策略：
  - 代理人电话直接显示真实号码
  - 农户用自己手机直接拨打（无中转）
  - 不使用隐私号服务

理由：
  ✅ 节省成本（隐私号每年约 5,000 元）
  ✅ 接通率更高（运营商不会拦截虚拟号）
  ✅ 简化技术接入（无需对接隐私号服务商）
  ✅ 农户操作更直接，符合菜农使用习惯
  ✅ 代理人是平台员工，号码暴露风险可控

风险与应对：
  风险 1：代理人被骚扰
    应对：代理人离职时统一更换号码 + 老号码做转接处理
    应对：高频骚扰用户可被代理人标记，触发账号审查

  风险 2：农户/代理人绕过平台私下交易
    应对：代理人 KPI 与平台订单挂钩，私下交易无收益
    应对：通话录音抽查（代理人入职协议授权）

  风险 3：号码隐私合规
    应对：代理人入职时签署"工作号码使用授权书"
    应对：用户协议中明确告知"代理人电话为工作号码"
```

#### X.4.7 通话核验机制（防跳过呼叫）

```
"转 AI"按钮的解锁条件：
  必须至少与一个代理人真实通话 > 0 秒

技术实现方案：

方案 A（推荐）：代理人后台自动弹窗 + 自动确认
  - 农户拨打代理人真实号码
  - 代理人手机响铃（无论接听与否）
  - [关键] 代理人后台同时自动弹出悬浮窗：
    ┌──────────────────────────────┐
    │ 📞 农户张三正在呼叫您          │
    │ 订单 #XXX 紧急调货             │
    │ 剩余时间：2h 30min             │
    │                              │
    │ 倒计时 3 秒后自动确认接通...    │
    │ [立即确认] [这不是我接的]       │
    └──────────────────────────────┘
  - 3 秒后自动确认（除非代理人点"这不是我接的"）
  - 自动解锁农户端"转 AI"按钮
  - 代理人接听后可在后台记录处理结果
  - 优势：跨平台、不依赖手机权限、不靠代理人记得点按钮
  - 劣势：代理人后台需长连接保持在线

方案 B（兜底）：Android 通话日志读取
  - App 申请通话日志权限
  - 农户拨打后回到 App，App 自动检测最近通话记录
  - 匹配代理人号码 + 通话时长 > 0 → 解锁
  - 优势：自动化、无需代理人操作
  - 劣势：iOS 不支持，且农户可能拒绝权限

方案 C（最终兜底）：30 分钟超时自动放行
  - 如果两种方案都失败，30 分钟后强制进入第二段
  - 但视为"未真实通话"，按"无故失联"处罚
  - 这条路径让流程不会卡死

第一期建议：方案 A + 方案 C 组合
  - 主路径走方案 A（代理人手动确认）
  - 异常路径走方案 C（超时强制流转 + 加重处罚）
  - 第二期视情况补充方案 B
```

### X.5 替代货差价分摊规则

#### X.5.1 场景 A：替代货比原价**贵**

```
差价计算：
  Δ = 替代价 - 原价

分摊方案：
  1. 优先从违约农户保证金扣除（直至扣完）
  2. 不足部分由平台垫付
  3. 平台垫付的金额计入农户"待追偿账户"
  4. 农户后续每笔订单分账时，自动扣 30% 偿还，直到清零

买家无需承担差价（这是平台给买家的兜底承诺）
```

#### X.5.2 场景 B：替代货比原价**便宜**

```
差价 Δ = 原价 - 替代价

分配方案：
  100% 退还买家（作为延误补偿）

理由：
  买家本来就因为延误受损（时间、备货计划），
  这部分差价是天然的补偿，不应由平台或新农户分享。
```

#### X.5.3 场景 C：等级不一致

```
情况 1：替代货等级低于原订单（如一级 → 二级）
  必须征得买家同意
  价格按"二级参考价"计算 → 差价退买家
  原违约农户额外扣信用 -10

情况 2：替代货等级高于原订单（如一级 → 特级）
  价格仍按原订单价（特级农户自愿低价销售）
  无需买家同意
  系统给替代农户额外补贴 20 元（鼓励高质量履约）
```

#### X.5.4 大幅差价的强制确认

```
当替代货任何一项偏离原订单超过以下阈值时，
必须买家二次确认（30 分钟内回复）：

- 价格偏离 > 15%
- 等级降低
- 送达时间延迟 > 4 小时
- 数量短缺 > 5%

买家不同意 → 自动转入退款流程
买家不回复 → 默认退款
```

### X.6 买家对称取消规则

#### X.6.1 设计原则

买家也必须有取消机制，但因为买家已付款，规则与农户略有不同：
- 农户违约 = 影响履约 → 重在保证金惩罚
- 买家违约 = 影响农户备货和市场库存 → 重在退款扣减

#### X.6.2 时间基准

```
T = 农户承诺的"送达市场时间"（与农户取消同基准）
```

#### X.6.3 买家取消惩罚阶梯

| 取消时机 | 信用扣分 | 退款比例 | 备注 |
|---|---|---|---|
| **> T - 24h** | -5 | 99%（扣 1% 手续费） | 农户尚未备货 |
| **T - 24h ~ T - 5h** | -15 | 95%（扣 5%） | 农户开始备货，需补偿 |
| **T - 5h ~ T** | -30 | 80%（扣 20%） | 农户接近发货，损失大 |
| **农户已送达市场后** | 禁止主动取消 | - | 只能走"拒收争议" |
| **不可抗力** | 0 | 100% | 凭证明 |

扣款分配（5% 案例）：
- 3% 给已备货的农户作为补偿
- 2% 平台运营成本

#### X.6.4 买家信用分

```
买家初始信用：100 分

权限影响：
  90 分以上：可享受"先发货后付款"模式（限老客户）
  70-90 分：正常付款流程
  50-70 分：必须 100% 预付款 + 限制大额订单
  < 50 分：暂停采购权 7 天
  < 20 分：永久封号
```

#### X.6.5 农户已送达后买家拒收的处理

```
农户已将货送到市场 → 买家临时不要 → 系统进入"代销流程"

处理方案：
1. 平台接收货物，进入"紧急代销池"
2. AI 自动撮合附近其他买家
3. 24 小时内售出 → 按实际售价结算给农户
4. 24 小时未售出 → 进入降价处理（每 6 小时降 5%）
5. 48 小时未售出 → 平台按底价收购，记为损耗

原买家承担：
  - 全部货款的 30% 作为违约金
  - 信用扣 -30
  - 进入"高风险买家"名单
```

### X.8 数据埋点与监控

为了持续优化规则，必须埋点以下数据：

| 指标 | 说明 | 用途 |
|---|---|---|
| 农户违约率 | 违约订单 / 总订单 | 评估保证金阶梯合理性 |
| 紧急调货成功率 | 三层兜底成功 / 触发次数 | 评估代理人和 AI 效果 |
| AI 调货成功率 | AI 层成功 / AI 触发次数 | 优化 AI 推送策略 |
| 外呼接通率 | 外呼接通 / 外呼次数 | 评估电话兜底有效性 |
| 平均兜底耗时 | 从违约到解决的平均时间 | 评估整体响应效率 |
| 平台垫付金额 | 月度垫付总额 | 评估财务风险 |
| 不可抗力申诉率 | 申诉数 / 违约数 | 防止申诉滥用 |
| 买家取消率 | 买家主动取消订单 / 总订单 | 评估买家行为 |

监控告警：
- 单日违约率 > 5% → 告警
- AI 调货成功率 < 60% → 告警
- 平台垫付月累 > 10,000 元 → 告警
