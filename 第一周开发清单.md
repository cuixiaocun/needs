## 一、数据库初始化（Day 1）

### 1.1 建表 SQL
```bash
# 在 MySQL 中执行以下脚本
mysql -u root -p needs_db < db_schema.sql
```

关键 8 张表（见 `db_schema.sql`）：
- `farmer_deposits` - 农户保证金账户
- `farmer_deposit_logs` - 保证金流水
- `emergency_dispatch_logs` - 紧急调货日志
- `agent_call_records` - 代理人呼叫记录
- `market_receiving_records` - 市场收货记录
- `market_dispatch_records` - 市场出货记录
- `farmer_settlements` - 农户结算单
- `settlement_items` - 结算单明细

### 1.2 复用旧表（从 v4.0）
- `users` - 用户基表
- `orders` - 订单表
- `order_items` - 订单明细
- `user_credits` - 用户信用积分
- 等其他（沿用 v4.0 数据库设计）

### 1.3 验证

### 2.3 数据库连接配置（config/database.config.ts）
```typescript
import { TypeOrmModuleOptions } from '@nestjs/typeorm';

export const databaseConfig: TypeOrmModuleOptions = {
  type: 'mysql',
  host: process.env.DB_HOST || 'localhost',
  port: parseInt(process.env.DB_PORT || '3306'),
  username: process.env.DB_USER || 'root',
  password: process.env.DB_PASSWORD || 'password',
  database: process.env.DB_NAME || 'needs_db',
  entities: ['src/modules/database/entities/**/*.entity.ts'],
  migrations: ['src/modules/database/migrations/**/*.ts'],
  synchronize: false,  // 生产禁用，使用 migrations
  logging: true,
};
```

## 八、常见坑 & 规避方案

| 坑 | 现象 | 解决 |
|-----|------|------|
| MySQL 时区错误 | `java.sql.SQLException: The server time zone value` | 设置 `?serverTimezone=UTC` |
| 支付宝签名失败 | 验签一直失败 | 确保私钥格式正确（PKCS8） |
| 货拉拉限流 | 频繁返回 429 | 做好重试 + 指数退避 |
| 紧急调货超时 | 第二段 AI 等待过长 | 按 R 时间动态调整（文档有规则） |
| 保证金扣除错误 | 余额变成负数 | 扣除前先检查可用余额 >= 扣除额 |
